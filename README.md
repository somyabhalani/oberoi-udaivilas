# The Oberoi Udaivilas – A Digital Discovery Journey

A high-fidelity, cinematic discovery experience for **The Oberoi Udaivilas**, Udaipur. This project focuses on a "Liquid Glass" design language, premium animations, and an immersive editorial narrative to showcase the luxury of one of the world's finest hotels.

## ✨ Key Features

- **Liquid Glass Aesthetic**: A modern, glass-morphism based UI with soft blurs, elegant overlays, and gold-leaf accents.
- **Cinematic Navigation**: A luxury "Liquid" navbar that adapts as you journey through the site.
- **Discovery Journey**: Smooth, scroll-triggered narratives covering:
  - **Accommodation**: High-end suite showcases with interactive galleries.
  - **Dining**: Visual culinary experiences at the Lakeside Pavilion.
  - **Experiences**: Curated guest activities from Astrology to Off-track adventures.
  - **Attractions**: A multi-destination guide to Udaipur's landmarks (City Palace, Nathdwara, etc.).
- **Signature Reveal Animations**: Hand-written GSAP signature reveals for section headers.
- **Deep Parallax**: Immersive vertical depth effects and zoom transitions on high-resolution assets.
- **Smooth Scroll Engine**: Powered by Lenis for a fluid, unhurried guest experience.

## 🛠️ Tech Stack

- **Core**: HTML5, Vanilla CSS3, JavaScript (ES6+)
- **Animation**: [GSAP](https://greensock.com/gsap/) (GreenSock Animation Platform)
- **Scroll Interaction**: [ScrollTrigger](https://greensock.com/scrolltrigger/)
- **Smooth Scrolling**: [Lenis](https://github.com/darkroomengineering/lenis)
- **Typography**: Inter, Playfair Display, and Pinyon Script (via Google Fonts)

## 🎨 Design Philosophy

The project aims to replicate the **unhurried luxury** of the Oberoi brand. Every interaction is designed to feel deliberate, smooth, and premium. The use of whitespace, editorial layouts, and gold-tinted typography creates a digital environment that feels as exclusive as the physical destination.

## 🚀 Deployment

The site is built as a static experience. To deploy:
1. Clone the repository.
2. Ensure the `public/` folder contains all necessary assets.
3. Host on any static provider (GitHub Pages, Vercel, Netlify).
